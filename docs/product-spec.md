# Product Spec

## 概要

`trasta` / `とらすた` は、ブログを中心に、作品・プロフィールを自然に閲覧できる個人ポートフォリオサイトです。

可愛い雰囲気はキャラクターを多用して作るのではなく、読みやすいレイアウト、小さな手描き素材、CSSアニメーション、控えめなアクセントカラーで表現します。

## ゴール

- ブログ記事を気持ちよく読めること。
- `about` と `works` に自然に移動できること。
- `trasta` / `とらすた` らしい可愛さと遊び心があること。
- Cloudflare Workers にデプロイできるモダンな構成であること。
- 将来的にログインして記事編集・新規投稿ができる余地を残すこと。

## ページ構成

### `/`

トップページです。

- `trasta` / `とらすた` の短い紹介を表示します。
- 最新ブログ記事を主役として表示します。
- `works` と `about` は下部または横の控えめなセクションに置きます。
- `tiger-peeking-wall-effects.png` などの素材を控えめに使います。

### `/blog`

ブログ一覧ページです。

- 記事一覧を新しい順に表示します。
- タグ・投稿日・更新日・読了時間を表示します。
- タグやカテゴリで絞り込める構造にします。
- 初期実装ではクライアント検索は必須ではありません。

### `/blog/$slug`

ブログ詳細ページです。

- Markdown/MDX本文を表示します。
- 目次を表示します。
- コードブロックは読みやすさを重視します。
- コードブロックごとにコピー機能を付けます。
- 共有ボタンを表示します。
- ページごとのOGPを出力します。
- X埋め込み、リンクカード埋め込みを利用できます。

### `/about`

プロフィールページです。

- `とらすた` の自己紹介、興味、使っている技術、活動リンクを表示します。
- キャラクター素材は署名や区切り程度に留めます。

### `/works`

作品一覧ページです。

- 作品カードを一覧表示します。
- 各作品にタイトル、説明、技術、リンク、作成時期を持たせます。
- 外部リンクは新規タブで開きます。

### `/works/$slug`

作品詳細ページです。

- 作品の背景、工夫、スクリーンショット、リンクを表示します。
- 初期実装では必須ではありませんが、データ構造は詳細ページを持てる形にします。

## ブログ機能

### 目次

- 見出し `h2` / `h3` から自動生成します。
- デスクトップでは本文横に追従表示します。
- モバイルでは記事冒頭に折りたたみ表示します。
- 現在読んでいる見出しをハイライトします。

### コードブロック

- シンタックスハイライトを有効にします。
- ファイル名・言語名を表示できるようにします。
- 横スクロール時も行が読みやすい余白を確保します。
- ダークモードでもコントラストを保ちます。
- 長い行でレイアウトが壊れないようにします。

### コピー機能

- コードブロックごとにコピー用ボタンを表示します。
- 成功時は `copied` 状態を短時間表示します。
- 失敗時はブラウザ標準の選択・コピーができる状態を維持します。

### 共有ボタン

- X共有ボタンを表示します。
- URLコピー用ボタンを表示します。
- 必要に応じて `navigator.share` を使います。
- 共有文言は記事タイトルとURLを基本にします。

### X埋め込み

- MDX内で専用コンポーネントを使えるようにします。
- 埋め込みが読み込めない場合も、元ポストへのリンクを表示します。
- レイアウトシフトを抑えるため、最低高さまたはプレースホルダーを用意します。

### リンク埋め込み

- MDX内でリンクカードコンポーネントを使えるようにします。
- タイトル、説明、URL、favicon、OG画像を表示できる形にします。
- 初期実装では手入力メタデータを優先し、自動取得は後回しにします。

### OGP

- 記事ごとに `title`、`description`、`publishedAt`、`updatedAt`、`tags`、`ogImage` を持ちます。
- `ogImage` が未指定の場合は、記事タイトルから動的OGP画像を生成します。
- OGP画像は `1200x630` を基本サイズにします。

## できれば欲しい機能

### adminログイン

Phase 2 の機能として設計します。

- `/admin/login` でログインします。
- ログイン後、記事一覧・新規投稿・編集画面を表示します。
- スマホでも本文編集しやすいUIにします。
- 下書き保存、公開/非公開、プレビューを扱える設計にします。

初期実装では、公開サイトとMarkdown/MDXコンテンツ管理を優先します。adminは設計だけ先に残し、MVP完成後に追加します。

- works一覧
- Markdown/MDX記事
- 目次
- コードブロックコピー
- 共有ボタン
- X埋め込み
- リンクカード埋め込み
- 記事ごとのOGP
- ライト/ダークモード
